# 偶发性错误

## SqlSugarClient 模式（超高性能）

每次请求 new 新实例，禁止存储于静态变量。

## 特别注意：
- 需设置 `IsAutoCloseConnection=true`
- 异步中确保每个 await 不遗漏（与 Dapper、EF Core 用法一致）
- 使用 `var newdb = db.CopyNew()` 保证线程安全
- 通过 IOC 创建新实例（对仓储更友好）

**异步知识：** `async void` / `async Action` 不受 await 等待，需注意线程安全；仅 `async Func<Task>` 有等待效果。

## 内部开线程或者 Job 需要在创建新实例保证线程安全有 2 种方式：
1. IOC 创建新实例（推荐）
2. `db.CopyNew` 适合【简单项目】和【只有个别方法报错或者报错频率不高懒的排查问题可以用这个】

> 注意： IsAutoCloseConnection=true 需要设置 true

## SqlSugarScope 模式（性能中上）

排查以下 3 条即可覆盖所有场景：

**排查1：** 异步方法 await 存在漏写，一个不能少。解决方案：找到漏掉的 await 建议补上遗漏的 await，如果不想补就用 `db.CopyNew()`。

**排查2：** 使用 await 返回值必须有 Task，不能是 Void（表达式中异步只能是 `Func<Task>` 不能是 `Action<X>`，比如 `List.ForEach` 就是 Action 要改普通 foreach）。建议改代码，如果不想修改 db 改成 `db.CopyNew()` 解决。

**排查3：** `Task.WhenAll`、`ParallelAsync` 和第三方定时任务库 Job（backgroundservice 也算）使用 `db.CopyNew()`。

## 方式一：db.CopyNew()

性能强大，等于原生 new SqlConnection 和 Dapper 原理一样。

```csharp
// db使用CopyNew()
await DB.CopyNew().Queryable<Order>().ToListAsync();

// 错误写法，因为CopyNew赋值给变量后，这个变量也要保证是线程安全的，尽量CopyNew在方法中使用
base.Conext = Db.CopyNew() // 错误写法

// 事务中要声名一个变量，CopyNew必须在当前方法使用
var newDb = DB.CopyNew();
newDb.BeginTran();
....
....
newDb.CommintTran();

// 仓储（请升级到5.1.4.78 之前版本有问题）
rep.CopyNew().Insert(p) // orm自带方法用这个
rep.CopyNew<自定义<类>>().MyXXX(p) // 用到自定义仓储方法用这个

// 多租户
// 5.1.4.106
var newDb = db.CopyNew();
// 正确
newDb.GetConnection("1")
// 错误:租户CopyNew不要用Scope
newDb.GetConnectionScope("1")
```

## 方式二：异步锁

点对点的线程安全写法，该功能一般适合用在 CopyNew() 不方便改的情况。

> 注意：
- 只能点对点的保证线程安全，就是这个接口单次请求不报错这个锁就有用
- 如果单次请求就报错那么这个锁就没有用
- 这个锁用的地方不要多（建议 5 个以下，只给特殊情况使用）
- 锁的性能有还是不错的，不过锁的实例是同一个，用的地方不能多

```csharp
using (await db.AsyncLock()) // 默认30秒超时
{
    // 里面await不能少
    await _Rep.InsertAsync(xx);
    await _Rep.InsertAsync(yy);
}
```

## 笨方法（100%可以解决）

以上方案覆盖绝大多数场景。若无法排查，使用笨方法。

> 禁止在公用方法中使用 `CopyNew()`，仅在当前业务方法中使用，否则事务难以维护。

## 异步注意事项总结

1. **async void 或者表达式的 async Action** 是不受 await 等待效果的，需要注意线程安全
2. **只有 async Func<Task>** 有等待效果
3. **List.ForEach 就是 Action**，要改普通 foreach
4. **Task.WhenAll** 和 **第三方定时任务库 Job**（backgroundservice 也算）使用 `db.CopyNew()`

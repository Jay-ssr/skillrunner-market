# 并发控制

## 并发累计（累加）

### 单条批量累计+1

```csharp
//正确写法：安全的字段累计
var result = db.Updateable<Student>()
    .SetColumns(it => it.Num == it.Num + 1)
    .Where(it => it.Id == 1)
    .ExecuteCommand();

//sql: num=num+1
```

### 批量更新并且字段+1

```csharp
var result = db.Updateable(updateObjs)
    .PublicSetColumns(it => it.Num, it => it.Num + 1)
    .ExecuteCommand();
```

### 批量更新并且字段+list中对应的值

```csharp
db.Updateable(list)
    .PublicSetColumns(it => it.Price, "+") //set price=price+list[i].price
    .ExecuteCommand();
```

## 防止提交覆盖，重复提交（乐观锁）

乐观锁适用条件：
1. 用户打开编辑界面，数据绑定到界面
2. 用户填写完成后提交保存

### 不依赖库同步（新功能）

```csharp
public class ULockEntity
{
    [SqlSugar.SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int Id { get; set; }
    public string Name { get; set; }

    [SqlSugar.SugarColumn(IsEnableUpdateVersionValidation = true)] //标识版本字段
    public long Ver { get; set; }
    //支持Guid long string DateTime (不推荐DateTime 时间有精度问题)
    //推荐用 string guid（使用long要序列化成string不然前端精度丢失）
}

//用法1：不扔错，Ver与数据库字段不同不报错返回0
var rows = db.Updateable(new ULockEntity() {
    Id = id,
    Name = "newname",
    Ver = 1551128313597136896
}).ExecuteCommandWithOptLock();

//用法2：扔出错误，Ver与数据库字段不同直接扔错误
var rows = db.Updateable(new ULockEntity() {
    Id = id,
    Name = "newname",
    Ver = 1551128313597136896
}).ExecuteCommandWithOptLock(true);

//try { ... } catch (VersionExceptions ex) { ... }
```

### 依赖库同步（老功能）

```csharp
public class StudentVersion
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int Id { get; set; }
    public string Name { get; set; }
    public DateTime CreateTime { get; set; }

    [SqlSugar.SugarColumn(
        IsEnableUpdateVersionValidation = true, //标识版本字段
        IsOnlyIgnoreInsert = true, //禁止插入
        IsOnlyIgnoreUpdate = true, //禁止更新
        ColumnDataType = "timestamp"
    )]
    public byte[] Timestamp { get; set; } //SqServer byte[] 其它数据库用 DateTime
}

db.Updateable(data).IsEnableUpdateVersionValidation().ExecuteCommand();
```

## 悲观锁的用法

悲观锁适用于读取和插入在同一事务中的场景，禁止两个请求同时操作同一数据。

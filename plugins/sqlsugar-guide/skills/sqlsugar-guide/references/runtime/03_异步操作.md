# 异步操作

## 异步作用

**作用**：释放主线程占用，提高吞吐量。直接使用并不能提高速度，在高并发、数据库请求耗时长的场景中使用。全异步写法无大碍，但会增加代码复杂度。

## async void 和 async Task 的区别

- **Task类型返回值**：有等待效果，既能保证线程安全又不会堵塞主线程，是推荐的用法。
- **void没有等待效果**：需要考虑线程安全，例如使用 `db.CopyNew()` 来保证。
- 对于表达式也有 Action 和 Func\<Task\>，其中 Action 等同于 void，Func\<Task\> 等同于 Task。
- List.ForEach 是一个 Action，所以 async 不会等待。可以替换为 foreach(var item in list)。

## 异步方法使用规则

SqlSugar中 `ToList` 是同步方法，那么 `ToListAsync` 这种是异步方法，通过 `Async` 来区别是同步还是异步方法。

## 返回值类型只能是 Task 或者 Task\<类型\>

```csharp
// 错误用法
Public void GetAll(){ 异步方法 }       // 方法要有Task返回值
Public async void GetAll(){ await 异步方法 }  // 方法要有Task返回值

// 正确用法
Public async Task GetAll(){ await 异步方法 }  // 正确
Public async Task<T> GetAll(){ await 异步方法; return xxx; }  // 正确
```

## 异步表达式需要看类型是不是Task

```csharp
// 错误：Action等于Void不是Task
Action a = async () => await 异步方法

// 正确：Func<Task> 等同于 Task
Func<Task> a = async () => await 异步方法

// 错误：Parallel.Foreach 参数是 Action，不支持异步委托
Parallel.Foreach(Action action)

// 错误：List.Foreach 参数是 Action，不支持异步委托
List.Foreach(Action action)

// 正确：foreach是C#语法不是函数，和委托没关系
public Task TestAsync()
{
    foreach(var item in list)
    {
        await 异步方法;
    }
}
```

## 正确的异步用法

```csharp
// 正确用法：每个异步方法都用await串起来
public async Task<List<Order>> Test()
{
    var db = GetInstance();
    await GetAsync();
    await db.Queryable<OrderItem>().ToListAsync();
    return await db.Queryable<Order>().ToListAsync();
}

// 正确用法：使用Task.Run包装
public void TestAsync()  // 唯一一种返回值不是Task的异步用法
{
    Task.Run(async () => {
        // 如果是 SqlSugarClient 需要 new
        await 异步方法;
    })
}

// 正确用法
public Task TestAsync()
{
    return Task.Run(async () => {
        await 异步方法;
    })
}
```

## 异步Canceltoken

```csharp
db.Ado.CancellationToken = token;
List<Order> list1 = await db.Queryable<Order>().ToListAsync();
List<Order> list2 = await db.Queryable<Order>().ToListAsync();
db.Ado.RemoveCancellationToken();
```

## 异步常见错误

## The connection does not support MultipleActiveResultSets

解决方案：
1. SqlSugarClient 替换成 SqlSugarScope，因为 SqlSugarScope 是线程安全对象代码容错率高
2. 异步用法错引起的，排查没用Await调用的异步方法

## Task.WhenAll代替Parallel

`Parallel.ForEach` 只能使用同步方法，不能使用异步方法：

```csharp
// Parallel.Forach 参数是 Action 而不是 Func<Task>，不支持异步表达式
public static ParallelLoopResult ForEach<TSource>(IEnumerable<TSource> source, Action<TSource> body);
```

请使用 Task.WhenAll 替换 Parallel，具体用法见 [并发执行](#7-并发执行)。

## 特殊场景解决方案

有些情况必须强制 new 一个新对象出来保证线程安全，SqlSugarScope 也需要这样处理。

### 不加Await用异步

> 注意：没有 await 一定要加 CopyNew (SqlSugarScope 也要加)。因为 db 对象是不支持跨上下文使用，直接写异步不加 await 就会出现在不同上下文使用 db。

```csharp
// 两种方式都必须加CopyNew()
db.CopyNew().Queryable<T>().ToListAsync();  // 事务中用 var mydb = db.CopyNew()
db.CopyNew().Queryable<T>().ToListAsync();
```

### 未知的第三方组件Job

```csharp
db.CopyNew().Queryable<T>().ToListAsync();
```

### Task.WhenAll 或者Parallel

```csharp
db.CopyNew().Queryable<T>().ToListAsync();
```

就这几种情况必须要加 `db.CopyNew()`，其他只要用法正确都OK。

## 两种方式强制线程安全

### CopyNew()

SqlSugarClient 和 SqlSugarScope 都能用（SqlSugarClient 模式推荐用 CreateScope）：

```csharp
db.CopyNew().Queryable<T>().ToListAsync();
```

### ServiceProvider.CreateScope

只支持 SqlSugarClient，也是 SqlSugarClient 推荐用法。

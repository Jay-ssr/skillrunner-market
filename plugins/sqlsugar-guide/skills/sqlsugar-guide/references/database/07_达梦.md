# 达梦

- **NuGet**: `SqlSugarCore` + `SqlSugarCore.Dm`

- **DbType**: `DbType.Dm`

## 连接字符串
```csharp
//老版本
PORT=5236;DATABASE=DAMENG;HOST=localhost;PASSWORD=SYSDBA;USER ID=SYSDBA

//新版本
Server=localhost;User Id=SYSDBA;PWD=SYSDBA;DATABASE=新DB

//带Schema
Server=153.101.101:5236;User Id=SYSDBA;PWD=123456;SCHEMA=myshcema;DATABASE=DAMENG
```

## 初始化
```csharp
SqlSugarClient db = new SqlSugarClient(new ConnectionConfig()
{
    DbType = DbType.Dm,
    ConnectionString = "Server=localhost;User Id=SYSDBA;PWD=SYSDBA;DATABASE=新DB;SCHEMA=myshcema",
    IsAutoCloseConnection = true
});
```

## 注意事项

### 1. 达梦表模式

默认为大写表模式。

存在小写字母时禁用自动转大写：
```csharp
MoreSettings = new ConnMoreSettings()
{
    IsAutoToUpper = false  //禁用自动转成大写表
}
```

### 2. Docker 版本 MySQL 模式

Docker 按官网安装可能装成 MySQL 模式，导致 ORACLE 分页无效：
```csharp
MoreSettings = new ConnMoreSettings()
{
    DatabaseModel = SqlSugar.DbType.MySql  //启用达梦mysql模式分页的兼容
}
```

### 3. 达梦 Clob、Text 类型用法

```csharp
[SugarColumn(SqlParameterDbType = typeof(NClobPropertyConvert))]
public string Name { get; set; }
```

clob（longtext）驱动有 BUG，部分用户出现插入空白。解决方案：
- 升级驱动：SqlSugarCore.Dm 1.3.0+
- 禁止 Vahrchar(36) 转 GUID

### 4. varchar36ToGuid 配置

```
varchar36ToGuid=TRUE 表示返回.net Guid 类型
varchar36ToGuid=FALSE 表示返回.net string 类型
```

### 5. 麒麟系统优化

程序启动时设置最小线程数：
```csharp
ThreadPool.SetMinThreads(250, 250);  //设置最小线程池为250，性能有提升
```

### 6. 常见错误

6.1 `.NET Framework 下面 dll 无效`
- 升级 SqlSugar

#### 6.2 大文件 clob 或者 blob 问题
- NuGet 独立安装 SqlSugarCore.Dm 最新版本

#### 6.3 存储过程无效
- 参数的顺序要一样

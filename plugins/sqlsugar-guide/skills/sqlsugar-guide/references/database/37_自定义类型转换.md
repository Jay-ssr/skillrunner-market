# 自定义类型转换

### 自定义类型（5.1.4.68）

SqlSugar 提供自定义类型以满足各种业务需求和数据库差异。

#### 自带的通用转换

自带转换出于数据安全和性能考虑不支持逆向转换（如 varchar 转 datetime）。

简单类型强制转换使用 ORM 自带通用转换。

```csharp
[SugarColumn(SqlParameterDbType = typeof(CommonPropertyConvert))] // CommonPropertyConvert ORM自带的
public DateTime DcValue { get; set; } // 5.1.4.62版本支持

// CommonPropertyConvert可以F12看他源码重新改一个新类完全可以自定义
```

自定义可 F12 查看源码，按此模式编写新类。

#### 自定义类示例

以下仅演示转换器定义方式，ORM 自带功能已包含。

```csharp
// 用例1：
public class DictionaryConvert : ISugarDataConverter
{
    public SugarParameter ParameterConverter<T>(object value, int i)
    {
        // 该功能 IsJson 可实现，此处仅作示例
        var name = "@myp" + i;
        var str = new SerializeService().SerializeObject(value);
        // 可以更改DbType = System.Data.DbType.XXX
        // PgSql中可以使用: CustomDbType = NpgsqlDbType.XXX;
        return new SugarParameter(name, str);
    }

    public T QueryConverter<T>(IDataRecord dr, int i)
    {
        // 打断点调试
        // 该功能 IsJson 可实现，此处仅作示例
        var str = dr.GetValue(i) + "";
        return new SerializeService().DeserializeObject<T>(str);
        // 如果不是序列化要转成T如下
        // (T)(object) str
    }
}
```

#### 使用自定义转换器

```csharp
[SugarColumn(ColumnDataType = "varchar(2000)", SqlParameterDbType = typeof(DictionaryConvert))]
public Dictionary<string, object> DcValue { get; set; } // 5.1.3.53-preview08
```

### 内置转换能力边界

SqlSugar 的类型处理分为"内置直接支持"和"自定义转换器"两档，以下是完整的适用范围。

#### 不需要自定义转换器的场景

| 场景 | 用法 | 原理 |
|------|------|------|
| 枚举存 int | `public SomeEnum Value { get; set; }` | ORM 自动 `Convert.ToInt64` 写入，读取时按属性枚举类型还原 |
| 枚举存 string | `[SugarColumn(SqlParameterDbType = typeof(EnumToStringConvert))]` | 内置 `EnumToStringConvert` |
| 基础类型数组（PostgreSQL） | `[SugarColumn(ColumnDataType = "text[]", IsArray = true)] public string[] Arr { get; set; }` | Npgsql 返回的类型与属性类型完全一致，`GetArray<T>` 直接 `(T)obj` 强转 |
| JSON 复杂类型 | `[SugarColumn(IsJson = true)]` | ORM 内置序列化/反序列化 |
| 简单类型强转 | `[SugarColumn(SqlParameterDbType = typeof(CommonPropertyConvert))]` | 内置通用转换，仅限简单类型强制转换（如 DateTime ↔ string），**不支持数组或元素级枚举转换** |

#### 自定义转换器的场景

**1. 枚举数组存为 PostgreSQL `integer[]`（最常见）**

```csharp
// ❌ 只靠 IsArray = true 不行
[SugarColumn(ColumnDataType = "integer[]", IsArray = true)]
public ApplicationType[] AppTypes { get; set; } // 读取时会抛 InvalidCastException

// ✅ 配合自定义转换器
[SugarColumn(ColumnDataType = "integer[]", IsArray = true,
    SqlParameterDbType = typeof(ApplicationTypeArrayConvert))]
public ApplicationType[] AppTypes { get; set; }
```

**原因（源码级）：** `IDataReaderEntityBuilder` 读取数组时调用 `GetArray<T>`（`IDataRecordExtensions`），该方法只做 `(T)obj` 直接强转。Npgsql 从 `integer[]` 返回 `int[]`，而 C# 不允许 `int[]` → `MyEnum[]` 强转，即使枚举底层是 int。写入路径的 `ArrayNull` 方法也只处理 `Guid[]`、`int[]`、`long[]`、`short[]`，枚举数组不在其中。

**2. 枚举数组转换器模板**

适用于任何 `enum[] → integer[]` 场景，无需为每种枚举单独写一个类：

```csharp
public sealed class EnumArrayConvert<TEnum> : ISugarDataConverter where TEnum : struct, Enum
{
    public SugarParameter ParameterConverter<T>(object columnValue, int columnIndex)
    {
        var name = "@EnumArr" + columnIndex;
        if (columnValue == null)
            return new SugarParameter(name, null, System.Data.DbType.Int32) { IsArray = true };

        var values = ((TEnum[])columnValue).Select(e => Convert.ToInt32(e)).ToArray();
        return new SugarParameter(name, values, System.Data.DbType.Int32) { IsArray = true };
    }

    public T QueryConverter<T>(IDataRecord dataRecord, int i)
    {
        var raw = dataRecord.GetValue(i);
        if (raw == DBNull.Value || raw is null)
            return default!;

        var ints = raw switch
        {
            int[] direct => direct,
            Array arr => arr.Cast<object>().Select(Convert.ToInt32).ToArray(),
            string text => ParseStringValues(text),
            _ => throw new InvalidOperationException(
                $"Unsupported array payload for {typeof(TEnum[]).Name}: {raw.GetType().FullName}")
        };

        var enums = ints.Select(v => (TEnum)(object)v).ToArray();
        return (T)(object)enums;
    }

    private static int[] ParseStringValues(string value)
    {
        var trimmed = value.Trim().TrimStart('{', '[').TrimEnd('}', ']');
        if (string.IsNullOrWhiteSpace(trimmed)) return [];
        return trimmed.Split(',', StringSplitOptions.RemoveEmptyEntries | StringSplitOptions.TrimEntries)
            .Select(s => Convert.ToInt32(s)).ToArray();
    }
}
```

使用：

```csharp
[SugarColumn(ColumnDataType = "integer[]", IsArray = true,
    SqlParameterDbType = typeof(EnumArrayConvert<ApplicationType>))]
public ApplicationType[] AppTypes { get; set; } = [ApplicationType.App];
```

### 判断速查

```
属性是数组？
├─ 元素类型是基础类型（string、int、long、Guid 等）
│   └─ IsArray = true 即可，不需要转换器
└─ 元素类型是枚举
    └─ 自定义转换器（枚举数组不在 ORM 内置处理范围内）

属性是单个值？
├─ 枚举存 int → 直接用，无需任何标注
├─ 枚举存 string → EnumToStringConvert（内置）
├─ 对象/集合存 JSON → IsJson = true（⚠️ 属性类型用对象，不用 string）
└─ 其他简单类型强转 → CommonPropertyConvert（内置）
```

### 现有类型支持

#### JSON类型

JSON 类型优先按本章前面的 `SugarColumn(IsJson = true)` 和自定义序列化方案处理。

#### 枚举类型

**int存储：** 直接使用。

```csharp
public DbType DcValue { get; set; }
```

**string存储：**

```csharp
// 如果枚举想存string
[SugarColumn(ColumnDataType = "varchar(20)", SqlParameterDbType = typeof(EnumToStringConvert))] // 这个类是ORM自带的
public DbType DcValue { get; set; }
```

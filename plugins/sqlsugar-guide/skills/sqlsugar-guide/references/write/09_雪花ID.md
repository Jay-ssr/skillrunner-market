# 雪花ID

### ORM中使用雪花ID

#### 普通插入

```csharp
public class 实体
{
    [SugarColumn(IsPrimaryKey = true)] //long类型的主键会自动赋值
    public long Id { get; set; }
    public string Name { get; set; }
}

long id = db.Insertable(实体).ExecuteReturnSnowflakeId(); //单条插入返回雪花ID
List<long> ids = db.Insertable(List<实体>).ExecuteReturnSnowflakeIdList(); //多条插入批量返回
```

#### 导航插入

导航操作中 `long` 主键未赋值时自动填充雪花ID。

#### 手动调用雪花ID

```csharp
var id = SnowFlakeSingle.Instance.NextId(); //也可以在程序中直接获取ID
```

### 雪花ID重复问题

使用雪花ID前设置 `WORKID`，每个不能共享 `SnowFlakeSingle` 的服务实例使用独立的 `WorkId`：

```csharp
//程序启动时执行一次就行
SnowFlakeSingle.WorkId = 唯一数字; //从配置文件读取不一样

//服务器时间修改后同步修改WorkId
```

> 参数说明：
> - workerId机器ID：2进制5位，31个
> - datacenterId机房ID：2进制5位，31个

### Javascript精度问题

#### 方式1：序列化成string

```csharp
[Newtonsoft.Json.JsonConverter(typeof(ValueToStringConverter))]
[SugarColumn(IsPrimaryKey = true)]
public long Id { get; set; }

//配置API使用JSON.NET
services.AddControllers().AddNewtonsoftJson(opt => {
    opt.SerializerSettings.ReferenceLoopHandling = ReferenceLoopHandling.Ignore;
    opt.SerializerSettings.ContractResolver = new DefaultContractResolver();
});
```

#### 方式2：使用短雪花ID

短雪花ID，需自行提供算法。

### 自定义雪花ID算法

```csharp
//程序启动时执行一次就行
StaticConfig.CustomSnowFlakeFunc = () => {
    return 你的雪花ID方法();
};
```

### ⚠️ 陷阱：配置 CustomSnowFlakeFunc 后的 API 选择

`CustomSnowFlakeFunc` **仅**被以下 API 调用：

| API | 触发雪花 | 说明 |
|-----|---------|------|
| `ExecuteReturnSnowflakeId(Async)()` | ✅ | 生成 → 插入 → 返回 ID |
| `ExecuteReturnSnowflakeIdList()` | ✅ | 批量版本 |
| 导航插入（`long` 主键未赋值时） | ✅ | 自动填充 |
| `ExecuteCommand(Async)()` | ❌ | 普通 insert，`Id` 保持默认值 `0` |
| `ExecuteReturnEntity()` | ❌ | 回填自增列，不触发雪花 |

```csharp
// ❌ 实体 Id = 0
await db.Insertable(entity).ExecuteCommandAsync();

// ✅ 正确用法
entity.Id = await db.Insertable(entity).ExecuteReturnSnowflakeIdAsync();

// ✅ 批量
List<long> ids = db.Insertable(list).ExecuteReturnSnowflakeIdList();
```

### 时间回退处理

```csharp
//程序启动时执行一次就行
var ran = new Random();
StaticConfig.CustomSnowFlakeTimeErrorFunc = () => {
    return ran.Next(16, 18); //出现时间回退使用临时算法插入
};
```

人为调整时间需重启服务或修改 `WorkId`，避免长时间使用自定义ID。

# 表格保存

将增、删、改变成一个 API 保存到数据库，常用于 Winform 等场景。

### 表格保存入门

#### 跟踪实现

适合 Winform 单对象操作场景。

```csharp
db.Tracking(getAll); //必须是List
getAll.RemoveAt(0); //删除记录
getAll.Last().Name = "a"; //修改
getAll.Add(new xxx() { ... }); //添加
db.GridSave(getAll).ExecuteCommand(); //删除+更新+插入
```

#### 手动对比实现

非单对象场景可手动指定新旧列表对比。

```csharp
//1.删除不存在newList中的记录
//2.然后newList存在更新不存在删除
db.GridSave(oldList, newList).ExecuteCommand();
```

### 性能说明

- list 1000条只要更新1条：跟踪方式性能好
- list 1000条要更新300条：手动对比性能好

原理：跟踪没办法做到update join，因为每条记录列的数量不同只能update set a; update set b...一条一条更新，而非跟踪方式可以用Update Join批量更新方式。

### 表格保存+导航

自动支持到二级导航。

```csharp
//通过跟踪实现
db.Tracking(getAll);
getAll.RemoveAt(0);
getAll.Last().Name = "a";

db.GridSave(getAll).IncludesAllFirstLayer().ExecuteCommand();

//不使用跟踪
db.GridSave(getAll, getAll2).IncludesAllFirstLayer().ExecuteCommand();

//排除不要更新导航
db.GridSave(getAll, getAll2).IncludesAllFirstLayer("Items").ExecuteCommand();
```

### 复杂应用

#### 复杂层级的导航

```csharp
//拿出要删除的
var deleteObjs = db.GridSave(oldList, newList).GetDeleteList();

//执行导航删除
db.DeleteNav(deleteObjs)
    .Include(z1 => z1.SchoolA).ThenInclude(z1 => z1.RoomList)
    .Include(z1 => z1.Books)
    .ExecuteCommand();

//执行导航更新+插入
db.UpdateNav(newList, new UpdateNavRootOptions() { IsInsertRoot = true })
    .Include(z1 => z1.SchoolA).ThenInclude(z1 => z1.RoomList)
    .Include(z1 => z1.Books)
    .ExecuteCommand();
```

#### 结合BulkCopy

```csharp
var deleteObjs = db.GridSave(oldList, newList).GetDeleteList();
db.Deleteable(deleteObjs).PageSize(1000).ExecuteCommand(); //高性能分页删除
db.Storageable(new Order()).PageSize(1000).ExecuteSqlBulkCopy(); //大数据插入或者更新
```

### 只插或者更新（不要删除）

```csharp
//普通插入或者更新
//导航插入或者更新
db.UpdateNav(newList, new UpdateNavRootOptions() { IsInsertRoot = true })
    .Include(z1 => z1.SchoolA).ThenInclude(z1 => z1.RoomList)
    .Include(z1 => z1.Books)
    .ExecuteCommand();
```

# 实体配置：内置特性

## 使用用例

常规 CRUD 只需正确配置主键和自增列；如果实体名或属性名与数据库不一致，再补充表名和列名映射。

### 主键 + 自增

```csharp
[SugarTable("dbstudent")] // 当实体名和表名不一致时，指定表名
public class Student
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)] // 数据库自增列才配置 IsIdentity
    public int Id { get; set; }
    public int? SchoolId { get; set; }
    [SugarColumn(ColumnName = "StudentName")] // 列名不一致时指定映射
    public string Name { get; set; }
}
```

### 多主键 / 复合主键 / 联合主键

字段是自增列才配置 `IsIdentity`；否则只标记 `IsPrimaryKey`。

```csharp
public class Student
{
    [SugarColumn(IsPrimaryKey = true)]
    public Guid Pk1 { get; set; }
    [SugarColumn(IsPrimaryKey = true)]
    public Guid Pk2 { get; set; }
    public string Name { get; set; }
}
```

### 无主键

表没有主键时，更新和删除都需要显式指定条件。

```csharp
public class Student
{
    public Guid Id { get; set; }
    public string Name { get; set; }
}
```

**结论**：实体配置应与数据库结构保持一致，重点是主键和自增列。

## 特性明细

以下是 CRUD 常用特性，不包含建表属性；建表相关请看 Code First 专题。

| 特性名 | 说明 |
|--------|------|
| IsIdentity | 自增列。Oracle12+ 按驱动能力启用；Oracle11 及以下配合 `OracleSequenceName` |
| IsPrimaryKey | 主键标记 |
| ColumnName | 数据库列名与实体属性名不一致时使用 |
| IsIgnore | ORM 完全忽略该列；常用于数据库不存在的属性。导航查询自动 `IsIgnore=true` |
| IsOnlyIgnoreInsert | 插入时忽略该列；常用于使用数据库默认值 |
| IsOnlyIgnoreUpdate | 更新时忽略该列 |
| InsertServerTime | 插入时使用数据库时间 |
| UpdateServerTime | 更新时使用数据库时间 |
| InsertSql | 插入时按 SQL 赋值。可写 `0`、`'a'`、`newid()`；5.1.4.151 起支持格式化，如 `数据库函数('{0}')` |
| UpdateSql | 更新时按 SQL 赋值。可写 `0`、`'a'`、`newid()`；5.1.4.151 起支持格式化，如 `数据库函数('{0}')` |
| QuerySql | 5.1.4.128+。单表查询且未使用 `Select` 时，按 SQL 生成列，例如 `Cast(Num_float64 as varchar(500)) AS Num_float64`。常用于驱动不支持的类型；联表或显式 `Select` 请改用函数或扩展函数 |
| OracleSequenceName | Oracle 序列；设置后该列等同于自增列 |
| MaxParameterNameLength | 一般设为 30；用于兼容 Oracle11 参数名和索引名超长 |

建表细节放在 Code First 专题，这里只保留 CRUD 常用属性。

# 连接池原理

本文介绍底层原理。理解连接池有助于判断线程安全和连接释放行为。

## ADO.NET 连接池

### 如何启用和禁用

在连接字符串中将 `Pooling` 设为 `true` 或 `yes` 即启用连接池；默认已启用。

| 状态 | Open | Close |
|------|------|-------|
| 启用连接池（默认） | 优先复用 `sleep` 状态连接；没有则创建新连接 | 把当前连接置为 `sleep` |
| 禁用连接池 | 打开新连接 | 关闭连接 |

启用后，系统会根据并发情况将连接保持在合理区间，不会立刻清空，而是把连接置为 `sleep` 状态。

### `Dispose()`

`Dispose()` 和 `Close()` 在连接层面做的是同一件事；`Dispose()` 会额外销毁当前 C# 对象。

```csharp
var db = GetInstance();
db.Open();
Console.WriteLine(db.Ado.Connection.GetHashCode());
db.Close();
Console.WriteLine(db.Ado.Connection.GetHashCode());
db.Dispose();
// 再次使用 db 时会 new 出新的 SqlConnection；这点和直接操作 ADO.NET 不同，ADO.NET 会直接报错
Console.WriteLine(db.Ado.Connection.GetHashCode());
```

| 操作 | 后续使用 |
|------|----------|
| `Dispose()` | 内部会换成新的对象 |
| `Close()` | 还是同一个 `db` 对象 |

## SqlConnection 和连接池

`SqlConnection` 不是线程安全对象，不同上下文应 `new` 不同实例。

```csharp
con.Open(); // 请求连接池中的连接；没有空闲连接则创建
con.Close(); // 将当前连接置为 sleep，长时间不用可能被销毁
```

| 要点 |
|------|
| 同一个对象多次 `Open` / `Close` 都是在同一连接池体系内操作 |
| 多个 `SqlConnection` 即使不同时请求，也可能复用同一连接池 |
| 并发请求时优先找空闲连接，没有再新建 |

.NET 对连接池封装已经很成熟，通常无需额外配置。

**连接超时**：在连接字符串中追加 `Connection Timeout=10`，默认 30 秒。

## SqlSugarClient 原理

### 手动释放模式

和 `SqlConnection` 原理基本一致，底层仍是 `SqlConnection`，同时要自行保证线程安全。

```csharp
SqlSugarClient db = new SqlSugarClient(new ConnectionConfig()
{
    ConnectionString = "Server=.xxxxx",
    DbType = DbType.SqlServer,
    IsAutoCloseConnection = false // 手动释放，属于长连接
});
// 需要手动 using
// 或手动 db.Open(); db.Close();
```

### 自动释放模式

推荐。无需手写 `using`、`open`、`close`。

```csharp
SqlSugarClient db = new SqlSugarClient(new ConnectionConfig()
{
    ConnectionString = "Server=.xxxxx",
    DbType = DbType.SqlServer,
    IsAutoCloseConnection = true // 自动释放
});

// 情况1：无事务时，每次操作自动 open / close
// 情况2：有事务时，开启事务时 open，提交或回滚时 close
```

## SqlSugarScope 原理

`SqlSugarScope` 是对 `SqlSugarClient` 的封装：
- 支持线程安全
- 不同上下文自动创建新的 `SqlSugarClient`
- 编码时通常不必手动关心线程安全

以下场景 `SqlSugarScope` 无法自动隔离上下文，需手动 `CopyNew()`：`Task.WhenAll`、`Parallel`、Job、异步代码未 `await`。

### 什么是上下文

- **异步**：同一串 `await` 视为同一上下文
- **同步**：同一线程视为同一上下文

同一个 `SqlSugarScope` 会在同一上下文共享对象，在不同上下文自动 `new` 新对象。

## 如何验证是否释放

未释放连接积累到一定数量就可能报错，不同数据库阈值可能有差异。

```csharp
for (int i = 0; i < 501; i++)
{
    SqlSugarClient db = new SqlSugarClient(new ConnectionConfig()
    {
        DbType = DbType.Oracle,
        ConnectionString = Config.ConnectionString,
        IsAutoCloseConnection = false // 设为 false 时这里会直接报错
    });

    db.Ado.GetInt("SELECT 1 from dual");
}
// IsAutoCloseConnection = true：执行成功
// IsAutoCloseConnection = false：可能出现连接池超时或超过上限等错误
```

## 总结

```text
SqlSugarScope  = 底层 + 自动释放 + 上下文安全
SqlSugarClient = 底层 + 自动释放控制
SqlConnection  = 底层
```

## Code First 入门

会根据连接字符串对应的数据库执行建库/建表。

```csharp
// 建库：不存在时创建
db.DbMaintenance.CreateDatabase();

// 建表：按实体创建
db.CodeFirst.InitTables(typeof(CodeFirstTable1));
```

## 注意点

- CodeFirst 适合快速开发；中后期应逐步禁用高风险自动变更
- 数据库账号需要较高权限
- SQLite 不支持删列和改列，只支持新增列
- 如果表不是 SqlSugar 创建的，直接执行 CodeFirst 可能报错

### 创建表&修改表

#### 纯特性建表（建表、批量建表）

默认类型支持多种数据库，能不设置尽量用自带的；`ColumnDataType`（想要多库兼容看 4.2 和 9）一般用于特殊情况。

```csharp
public class CodeFirstTable1
{
    [SugarColumn(IsIdentity = true, IsPrimaryKey = true)]
    public int Id { get; set; }
    public string Name { get; set; }
    // ColumnDataType 一般用于单个库数据库，如果多库不建议用
    [SugarColumn(ColumnDataType = "Nvarchar(255)")]
    public string Text { get; set; }
    [SugarColumn(IsNullable = true)] // 可以为NULL
    public DateTime CreateTime { get; set; }
}

/*** 创建单个表 ***/
db.CodeFirst.SetStringDefaultLength(200).InitTables(typeof(CodeFirstTable1));

/*** 手动建多个表 ***/
db.CodeFirst.SetStringDefaultLength(200)
    .InitTables(typeof(CodeFirstTable1), typeof(CodeFirstTable2));

/*** 批量创建表 ***/
// 语法1：
Type[] types = Assembly
    .LoadFrom("XXX.dll") // 如果.dll报错，可以换成xxx.exe 有些生成的是exe
    .GetTypes().Where(it => it.FullName.Contains("OrmTest.")) // 命名空间过滤，可以写其他条件
    .ToArray(); // 断点调试一下是不是需要的Type，不是需要的在进行过滤
db.CodeFirst.SetStringDefaultLength(200).InitTables(types);

// 语法2：
Type[] types = typeof(任意实体类中的类).Assembly.GetTypes()
    .Where(it => it.FullName.Contains("OrmTest.")) // 命名空间过滤，你也可以写其他条件过滤
    .ToArray();
db.CodeFirst.SetStringDefaultLength(200).InitTables(types);
```

#### 建表技巧：自动Nullable

上面的例子需要加 `IsNullable=true` 才能创建可空类型，否则是必填。通过实体 AOP 配置自动处理 `int?` / `decimal?`：

```csharp
var db = new SqlSugarClient(new ConnectionConfig()
{
    DbType = SqlSugar.DbType.SqlServer,
    ConnectionString = Config.ConnectionString,
    IsAutoCloseConnection = true,
    ConfigureExternalServices = new ConfigureExternalServices
    {
        // 注意：此处 AOP 设置不能省略
        EntityService = (c, p) =>
        {
            /*** 低版本C#写法 ***/
            // int? decimal? 这种 IsNullable=true 不支持 string（下面 .NET 7 支持）
            if (p.IsPrimarykey == false && c.PropertyType.IsGenericType &&
                c.PropertyType.GetGenericTypeDefinition() == typeof(Nullable<>))
            {
                p.IsNullable = true;
            }

            /*** 高版C#写法 ***/
            // 支持 string? 和 string
            if (p.IsPrimarykey == false && new NullabilityInfoContext()
                .Create(c).WriteState is NullabilityState.Nullable)
            {
                p.IsNullable = true;
            }
        }
    }
});

// 定义CodeFirst实体
public class xxxxx1
{
    // 没有 ? 是必填
    [SugarColumn(IsIdentity = true, IsPrimaryKey = true)]
    public int id { get; set; }
    // 老版本 .NET 字符串无法加 ?，改用特性 [SugarColumn(IsNullable = true)]
    // 如果不想用 SugarColumn，在 AOP 中打断点确认 P.IsNullable=true 即成功
    public string RequiredName { get; set; }
    // 带 ? 可空
    public int? id1 { get; set; }
}
```

#### 无特性建表

实体类干净可用无特性写法；使用特性更方便：

```csharp
SqlSugarClient db = new SqlSugarClient(new ConnectionConfig()
{
    DbType = DbType.SqlServer,
    ConnectionString = Config.ConnectionString3,
    InitKeyType = InitKeyType.Attribute,
    IsAutoCloseConnection = true,
    ConfigureExternalServices = new ConfigureExternalServices()
    {
        EntityService = (s, p) =>
        {
            // 如果是Order实体进行相关配置
            p.IfTable<Order>()
                .UpdateProperty(it => it.id, it =>
                {
                    it.IsIdentity = true;
                    it.IsPrimarykey = true;
                })
                .UpdateProperty(it => it.Name, it =>
                {
                    it.Length = 100;
                    it.IsNullable = true;
                })
                .OneToOne(it => it.Item, nameof(Order.ItemId));

            // 如果Custom实体进行相关配置
            p.IfTable<Custom>()
                .UpdateProperty(it => it.id, it =>
                {
                    it.IsIdentity = true;
                    it.IsPrimarykey = true;
                })
                .UpdateProperty(it => it.Text, it =>
                {
                    it.DataType = StaticConfig.CodeFirst_BigString; // 支持多库的MaxString用法
                });

            // 可结合全局逻辑；有冲突时后面的覆盖前面的
            // 统一设置 nullable 等于 IsNullable=true
            // 低版本 C# 需手动写完整语法
            if (p.IsPrimaryKey == false && new NullabilityInfoContext()
                .Create(s).WriteState is NullabilityState.Nullable)
            {
                p.IsNullable = true;
            }
        }
    }
});

// 性能说明：EntityService 对同一实体只执行一次，通常不必担心性能
```

#### 修改表

修改实体后重新执行 CodeFirst 即可同步表结构。

注意：正式数据禁用删列操作；实体和数据库不同步时直接跑 Code First 可能误删列。

| 功能 | 说明 |
|------|------|
| 添加列 | 在实体加添加属性 |
| 修改列 | 3种方式：<br>1、删旧列后重建新列（兼容性好，但数据会丢失）<br>2、更改类型、非空、长度等后重新执行 Code First；如需调整 `decimal` / `double` 精度，可开启 `EnableCodeFirstUpdatePrecision = true`<br>3、通过 `OldColumnName` 改列名，尽量保留原数据 |
| 删除列 | 在实体中删除属性；生产环境建议先禁用删列再手动处理 |
| 索引 | 通过特性定义索引会自动创建；删除索引仍需手动处理 |
| 描述 | 表和列都支持备注 |
| 高安全级别：禁用更新 | 禁用后表结构不会自动变更，适合正式环境 |
| 高安全级别：手动对比迁移 | 先比对差异，再手动执行迁移，适合高安全要求场景 |

### SugarColumn API

#### 特性列表

| 名称 | 描述 |
|------|------|
| IsIdentity | 是否创建自增标识 |
| IsPrimaryKey | 是否创建主键标识 |
| ColumnName | 创建数据库字段的名称（默认取实体类属性名称） |
| ColumnDataType | 创建数据库字段类型<br>用法1：直接写 "varchar(20)"，不需要再设置长度<br>用法2：不设置时按 C# 类型自动推断<br>用法3：多库兼容场景下可配合自定义类型映射使用 |
| IsIgnore | ORM不处理该列 |
| ColumnDescription | 备注 表注释<br>新版本支持XML文件, XML名字不要自已改要和类库名一样<br>把xml发布dll相同目录 |
| Length | 长度 设成10会生成 xxx类型(10), 没括号的不设置 |
| IsNullable | 是否可以为null默认为false |
| DecimalDigits | 精度 如 decimal(18,2) length=18,DecimalDigits=2 |
| OracleSequenceName | 设置Oracle序列，设置后该列等同于自增列 |
| OldColumnName | 修改列名用，这样不会新增或者删除列 |
| IndexGroupNameList | 已弃用，改用 `[SugarIndex]` 特性 |
| UniqueGroupNameList | 已弃用，改用 `[SugarIndex]` 特性配合唯一约束 |
| DefaultValue | 用来建表设置字段默认值 |
| IsOnlyIgnoreInsert | 插入数据时取默认值 |

**注意：有2个属性用处不同**

- DefaultValue = 默认值 用来建表设置字段默认值
- IsOnlyIgnoreInsert = true 插入数据时取默认值

很多情况需要2个一起使用：
- 如果只建表不插入数据用1个
- 如果建表并且插入数据用2个

| ExtendedAttribute | 扩展属性，可以结合EntityService实体AOP使用<br>参考：https://www.donet5.com/ask/9/20054 |

#### 多库支持

大量使用 `ColumnDataType="nvarchar(50)"` 等自定义类型将无法很好支持多库（很多库不支持 nvarchar）。

也可通过 AOP 替换 `ColumnDataType` 实现多库兼容，自带的方案。

| C#类型 | 数据库类型 | 说明 |
|--------|-----------|------|
| string 大文本 | 5.1.3.44-preview06 推荐 | `[SugarColumn(ColumnDataType = StaticConfig.CodeFirst_BigString)]` |
| string 设置长度的字符串 | `[SugarColumn(Length = 10)]` | `public string FieldName { get; set; }` |
| int 整数 | 自动 | `public int FieldName { get; set; }` |
| short 整数小 | 自动 | `public short FieldName { get; set; }` |
| long 大数字 | 自动 | `public long FieldName { get; set; }` |
| bool 真假 | 自动 | `public bool FieldName { get; set; }` |
| decimal 默认 | 自动 | `public decimal FieldName { get; set; }` |
| decimal 自定义 | 18,2、18,4、18,6这几种兼容性好 | `[SugarColumn(Length = 18, DecimalDigits = 2)]` |
| DateTime 时间 | 自动 | `public DateTime FieldName { get; set; }` |
| 枚举 | 数据库存int | `public 枚举 FieldName { get; set; }` |
| byte[] 二进制 | 自动 | `public byte[] FileInfo { get; set; }` |

建议升级到 SqlSugarCore 5.1.3.46-preview09 及以上，多库支持更完善。

SqlServer 特殊配置：默认使用 Nvarchar（其他库用 varchar），可通过配置区分：

```csharp
DbType = SqlSugar.DbType.SqlServer,
ConnectionString = "字符串",
IsAutoCloseConnection = true,
MoreSettings = new ConnMoreSettings()
{
    SqlServerCodeFirstNvarchar = true,
}
```

注意：默认索引定义能力不足时，改用 `DbMaintenance` 或手动脚本处理。

```csharp
    public string Text { get; set; }

    public DateTime CreateTime { get; set; }
}
```


### 重要功能

#### 修改列名

注意1：属性随意命名不按规范会导致删列后重建。

注意2：SQLite 不支持修改列操作。

注意3：表特性如有禁止修改需先去掉。

```csharp

```

#### 添加表描述

不写特性也可用 XML 自动读取备注（XML 文件名和类库名一致）。

```csharp
db.CodeFirst.InitTables(typeof(CodeFirstTable));

[SugarTable("CodeFirstTable2", TableDescription = "表备注")] // 表添加备注
public class CodeFirstTable
{
    [SugarColumn(IsPrimaryKey = true, ColumnDescription = "主键")] // 列添加备注
    public Guid Id { get; set; }
}

// 技巧：表名可以为 null，取实体名
[SugarTable(null, TableDescription = "表备注")] // 表添加备注
```

**对性能要求高时可禁用 XML 方式读取备注：**

默认先读特性，没有备注再读 XML。

```csharp
IsAutoCloseConnection = true,
DbType = DbType.SqlServer,
ConnectionString = Connection,
LanguageType = LanguageType.Default,
MoreSettings = new ConnMoreSettings()
{
    // 禁用 XML 可提升首次初始化性能
    IsNoReadXmlDescription = true // 禁止读取XML中备注,true是禁用
}

// 禁用 XML 后只能通过特性写备注
[SugarTable("CodeFirstTable2", TableDescription = "表备注")]
[SugarColumn(ColumnDescription = "主键")] // 列添加备注
```

### 禁止删除列

注意：SQLite 本身不支持删列，可不设置。

实体类中删除属性后执行 `InitTables` 会同步删除数据库列。为避免误操作，支持禁止删列。

```csharp
[SugarTable("Custom", "客户", IsDisabledDelete = true)]
public class Custom
{
    public int Id { get; set; }
    public string Name { get; set; }
}
```

**全局AOP全部禁止：**

```csharp
DbType = SqlSugar.DbType.PostgreSQL,
ConnectionString = Config.ConnectionString,
ConfigureExternalServices = new ConfigureExternalServices()
{
    EntityNameService = (type, entity) =>
    {
        entity.IsDisabledDelete = true;
    }
}
```

### 禁止更新+删除

设置后 CodeFirst 不再检查实体与数据库差异，也不会更新数据库，可大幅提升执行性能。

`IsDisabledUpdateAll = true` 表示表存在时不执行任何更新操作。

```csharp
[SugarTable("Custom", "客户", IsDisabledUpdateAll = true)] // 安全级别比IsDisabledDelete更高，只创建不修改和删除
public class Custom
{
    public int Id { get; set; }
    public string Name { get; set; }
}

db.CodeFirst.SetStringDefaultLength(200).InitTables(typeof(Custom)); // 注意这行代码加的

// 上面配置等同于
// if (表不存在)
// {
//     db.CodeFirst.SetStringDefaultLength(200).InitTables(typeof(Custom));
// }
```

### 更新其他来源的表

默认会报错，阻止更新非 SqlSugar 创建的表。2 种解决方式：

1. 尝试删除非 SqlSugar 建的表主键约束再试；如果仍不行且无重要数据，删除表后重建。
2. 禁用更新该表。

```csharp
[SugarTable("Custom", "客户", IsDisabledUpdateAll = true)] // IsDisabledUpdateAll 表示不更新此表
```

### 自定义类型多库兼容

**注意：默认类型映射满足需求时无需 AOP 动态处理。**

使用自定义类型时需如下处理：

```csharp

            {
                p.DataType = "longtext";
            }
        }
    }
});
```

### 结构对比手动迁移

适合成熟、线上稳定的项目，不自动修改表结构，而是手工处理。

5.0.5.8 preview02 新增。

```csharp
Type[] types = xxxx;
var diffString = db.CodeFirst.GetDifferenceTables(types).ToDiffString();
```

通过结构对比拿到差异信息后，手动在数据库中执行变更。

**数据安全多重保险：**

定义 2 个接口：一个调用对比方法，一个调用更新方法。

```csharp
var diffString = db.CodeFirst.GetDifferenceTables(types).ToDiffString()
```

第二个接口调用更新：

```csharp
db.CodeFirst.InitTables(types);
```

更新前先调用对比接口确认变更符合要求，再执行更新接口。

### 设置创建表排序

不设置排序时按实体类顺序创建表，特殊情况可手动排序。

**规则说明：**

1. 主键列始终排第 1 位
2. 不设置默认为 0
3. 排在前面可设负数（如 -1），默认为 0

5.0.9.6版本：

```csharp
[SqlSugar.SugarTable("Custom", IsCreateTableFiledSort = true)] // 开启排序不能少
public class Custom
{
    [SqlSugar.SugarColumn(CreateTableFieldSort = 1)]
    public int Id { get; set; }
    [SqlSugar.SugarColumn(CreateTableFieldSort = 2)]
    public string Name { get; set; }
}

// 注意：默认是0，可以用-1排前面
```

**获取表名：**

```csharp
// 1. 不缓存
var list = GetTableInfoList(false);

// 2. 缓存不变的
var list = GetTableInfoList(true);

// 3. 不缓存+重写SQL比如:我只要MyTable开头的表
// 需要升级到：5.1.4.151+
var list = db.DbMaintenance.GetTableInfoList((dbtype, sql) =>
{
    if (dbtype == SqlSugar.DbType.SqlServer)
    {
        return sql.Replace("需要替换部分", "替换后部分");
    }
    return sql;
});
```

**获取列信息根据表名：**

```csharp
// 1. 不缓存
var list = db.DbMaintenance.GetColumnInfosByTableName("mytable", false);

// 2. 缓存不变的
var list = db.DbMaintenance.GetColumnInfosByTableName("mytable", true);

// 需要升级到：5.1.4.151+
var list = db.DbMaintenance.GetColumnInfosByTableName("mytable", (dbtype, sql) =>
{
    if (dbtype == SqlSugar.DbType.SqlServer)
    {
        return sql.Replace("需要替换部分", "替换后部分");
    }
    return sql;
});
```

# SqlSugar 接口体系与扩展点

> 基于源码 Interface/ 目录的 29 个接口，帮助理解整体设计、查找扩展入口。

## 接口总览图

```
ISqlSugarClient (聚合入口)
  ├─ IAdo                    底层 ADO.NET 操作、连接管理、事务
  ├─ ICodeFirst              CodeFirst 自动建表/迁移
  ├─ IDbFirst                DbFirst 反向生成实体
  ├─ IDbMaintenance          数据库结构维护（DDL）
  ├─ ITenant                 多租户/多库管理
  ├─ IFilter                 全局查询过滤器
  ├─ ISqlBuilder             SQL 生成引擎（各数据库方言）
  ├─ ILambdaExpressions      Lambda 表达式 → SQL 解析
  ├─ IContextMethods         上下文工具方法（序列化、转换、分页）
  ├─ IDbBind                 数据库类型 ↔ C# 类型映射
  │
  ├─ ISugarQueryable<T>      查询（支持 1~12 表泛型）
  ├─ IInsertable<T>          插入
  │    ├─ ISubInsertable<T>        子表级联插入
  │    └─ IParameterInsertable<T>  参数化插入
  ├─ IUpdateable<T>          更新（支持 1~4 表）
  ├─ IDeleteable<T>          删除
  ├─ ISaveable<T>            保存（已废弃 → IStorageable）
  ├─ IStorageable<T>         差异化存储（SplitInsert/Update/Delete）
  ├─ IFastest<T>             大数据量高性能操作（BulkCopy）
  ├─ IReportable<T>          报表查询
  │
  ├─ ISimpleClient<T>        简化仓储 CRUD
  │    └─ ISugarRepository        仓储标记接口
  │
  ├─ IDMLBuilder             DML 构建器基类
  ├─ IFastBuilder            快速操作内部构建器
  ├─ ICustomConditionalFunc  自定义条件表达式扩展点
  └─ ISugarDataConverter     自定义数据类型转换器
```

## 核心入口接口

### ISqlSugarClient

SqlSugar 的顶层客户端接口，聚合了所有子系统。由 `SqlSugarClient` 和 `SqlSugarScope` 实现。

关键属性：`Ado`, `CodeFirst`, `DbFirst`, `DbMaintenance`, `EntityMaintenance`, `QueryFilter`, `Aop`, `Context`, `CurrentConnectionConfig`, `TempItems`

## 查询接口

### ISugarQueryable\<T\>（含 T2~T12 多表变体）

单表到最多 12 表联查的 Fluent API 接口。多表通过泛型递进继承：

```csharp
ISugarQueryable<T,T2> : ISugarQueryable<T>
ISugarQueryable<T,T2,T3> : ISugarQueryable<T>
// ... 直到 T12
```

| 方法类别 | 关键方法 |
|----------|----------|
| 联表 | `LeftJoin`, `InnerJoin`, `RightJoin`, `FullJoin` |
| 导航联表 | `IncludeLeftJoin`, `IncludeInnerJoin` 等 |
| 条件 | `Where`, `WhereIF`, `Having`, `HavingIF`, `In` |
| 排序/分组 | `OrderBy`, `OrderByDescending`, `GroupBy`, `PartitionBy` |
| 投影 | `Select<TResult>`, `SelectMergeTable` |
| 分页 | `Skip`, `Take`, `ToPageList`, `ToOffsetPage` |
| 终结 | `ToList`, `First`, `Single`, `InSingle`, `Count`, `Max`, `Min`, `Sum`, `Avg` |
| 树形 | `ToTree`, `ToChildList`, `ToParentList` |
| 转换 | `ToDataTable`, `ToJson`, `ToDictionary`, `ToSql` |
| 行转列 | `ToPivotTable`, `ToPivotList`, `ToPivotJson` |
| 高级 | `WithCache`, `SplitTable`, `Mapper`, `TranLock`, `ForEach`, `GetAsyncEnumerable` |

### IReportable\<T\>

报表数据源转可查询接口：
- `ToQueryable()` — 返回 `ISugarQueryable<T>`
- `ToQueryable<Y>()` — 单列实体投影

## 写入接口

### IInsertable\<T\>

| 方法 | 用途 |
|------|------|
| `ExecuteCommand` / `ExecuteReturnIdentity` / `ExecuteReturnBigIdentity` | 执行并返回影响行数/自增ID |
| `ExecuteReturnSnowflakeId` / `ExecuteReturnSnowflakeIdList` | 返回雪花ID |
| `ExecuteReturnEntity` | 返回完整实体 |
| `InsertColumns` / `IgnoreColumns` | 列筛选 |
| `AddSubList` | 子表级联插入 |
| `UseParameter` | 强制参数化 |
| `EnableDiffLogEvent` | 审计日志 |
| `UseSqlServer` / `UseMySql` / `UseOracle` | 数据库特定批量 |
| `MySqlIgnore` / `PostgreSQLConflictNothing` | 冲突策略 |

### ISubInsertable\<T\>

父子表级联插入，支持树形子表递归。

### IUpdateable\<T\>（含 T2~T4 多表变体）

| 方法 | 用途 |
|------|------|
| `SetColumns` | 表达式更新列值 |
| `UpdateColumns` / `IgnoreColumns` | 列筛选 |
| `WhereColumns` | 按列匹配更新条件 |
| `ReSetValue` | 批量重新设值 |
| `InnerJoin<T2>` | 多表联表更新 |
| `ExecuteCommandWithOptLock` | 乐观锁更新 |
| `IsEnableUpdateVersionValidation` | 版本验证 |

### IDeleteable\<T\>

| 方法 | 用途 |
|------|------|
| `Where` / `WhereColumns` / `In` | 条件删除 |
| `IsLogic` | 逻辑删除 |
| `EnableQueryFilter` | 启用查询过滤器 |

### ISaveable\<T\>（已废弃）

> `[Obsolete("use Storageable")]` — 使用 `IStorageable<T>` 替代。

### IStorageable\<T\>

差异化存储（大数据保存），支持五种拆分：

| 方法 | 用途 |
|------|------|
| `SplitInsert` | 不存在的新数据 |
| `SplitUpdate` | 需要更新的数据 |
| `SplitDelete` | 需要删除的数据 |
| `SplitIgnore` | 忽略的数据 |
| `SplitError` | 有问题的数据 |
| `ToStorage` | 计算差异结果 |
| `WhereColumns` | 匹配条件列 |
| `TranLock` | 事务锁 |
| `ExecuteSqlBulkCopy` | BulkCopy 执行 |

### IFastest\<T\>

大数据量高性能操作：

| 方法 | 用途 |
|------|------|
| `BulkCopy` | 数据库原生批量插入 |
| `BulkUpdate` | 批量更新（指定条件列和更新列） |
| `BulkMerge` | 批量合并（Upsert） |
| `PageSize` / `BatchSize` | 分批控制 |

## 基础设施接口

### IAdo

底层 ADO.NET 抽象层：
- **连接管理**: `Open`, `Close`, `OpenAlways`, `CheckConnection`
- **事务**: `BeginTran`, `CommitTran`, `RollbackTran`, `UseTran`
- **原始查询**: `GetDataTable`, `GetDataSetAll`, `GetDataReader`, `GetScalar`
- **泛型查询**: `SqlQuery<T>`, `SqlQuery<T,T2...T7>`（多结果集）
- **标量**: `GetString`, `GetInt`, `GetLong`, `GetDouble`, `GetDecimal`, `GetDateTime`
- **性能统计**: `SqlExecutionTime`, `ConnectionExecutionTime`, `SqlExecuteCount`

### ISqlBuilder（partial interface）

SQL 生成引擎，各数据库方言实现：
- **标识符转义**: `GetTranslationTableName`, `GetTranslationColumnName`
- **条件 SQL**: `AppendWhereOrAnd`, `GetWhere`, `ConditionalModelToSql`
- **SQL 关键字**: `SqlFalse`, `SqlDateNow`, `SqlParameterKeyWord`
- **UNION**: `GetUnionAllSql`, `GetUnionSql`

### ILambdaExpressions（partial interface）

Lambda 表达式解析器：
- `Resolve(expression, resolveType)` — 核心解析方法
- `MappingColumns` / `MappingTables` — 运行时映射覆盖
- `SqlFuncServices` — 自定义 SQL 函数注册

### IContextMethods

上下文辅助工具：
- `DataReaderToList<T>` — DataReader → 实体列表
- `DataTableToList<T>` / `ListToDataTable<T>` — 互转
- `SerializeObject` / `DeserializeObject<T>` — 序列化
- `PageEach<T>` — 分批遍历
- `ToTree<T>` — 扁平列表转树形
- `ConditionalModelsToSql` — 条件模型 → SQL
- `EscapeLikeValue` — LIKE 转义

### IDbBind

类型映射绑定：
- `GetPropertyTypeName` — DB 类型名 → C# 类型名
- `GetCsharpTypeName` — 同上
- `GetDbTypeName` — C# 类型 → DB 类型
- `MappingTypes` — 映射列表
- `GuidThrow` / `IntThrow` / ... — 可抛异常的类型列表

### IDbMaintenance（partial interface）

数据库结构维护（DDL 操作）：
- **元数据查询**: `GetTableInfoList`, `GetColumnInfosByTableName`, `GetIndexList`
- **存在性检查**: `IsAnyTable`, `IsAnyColumn`, `IsPrimaryKey`, `IsIdentity`
- **DDL**: `CreateTable`, `AddColumn`, `UpdateColumn`, `DropColumn`, `CreateIndex`
- **备注**: `AddRemark`, `AddTableRemark`, `AddColumnRemark`
- **备份**: `BackupDataBase`, `BackupTable`

### ICodeFirst（partial interface）

CodeFirst 接口：
- `InitTables<T>()` — 按类型初始化表（支持 T1~T5）
- `InitTables(string namespace)` — 按命名空间初始化
- `GetDifferenceTables<T>()` — 获取表差异
- `SplitTables()` — 分表 CodeFirst
- `SetStringDefaultLength` — 字符串默认长度

### IDbFirst（partial interface）

DbFirst 接口（反向工程）：
- `CreateClassFile` — 生成实体类文件
- `SettingClassTemplate` / `SettingPropertyTemplate` — 自定义模板
- `UseRazorAnalysis` — Razor 引擎
- `Where` / `WhereColumns` — 过滤表/列

## 扩展/辅助接口

### ISimpleClient\<T\>

简化仓储接口，提供 `Change<T>()` 多态切换、`CopyNew()` 复制实例，以及全部 CRUD 方法（同步 + Async + CancellationToken）。

### ISugarRepository

仓储标记接口，仅持有 `Context` 属性。用于 DI 泛型约束。

### ITenant

多租户接口：
- `ChangeDatabase` — 切换数据库
- `AddConnection` / `RemoveConnection` — 管理连接
- `QueryableWithAttr<T>` — 按特性自动路由查询
- `InsertableWithAttr<T>` / `UpdateableWithAttr<T>` / `DeleteableWithAttr<T>` — 按特性操作
- `BeginTran` / `CommitTran` / `RollbackTran` — 跨库事务

### IFilter

全局查询过滤器：
- `Add(SqlFilterItem)` — 添加过滤器
- `Remove(string)` — 移除过滤器

### IDMLBuilder

DML 构建器基类（Insert/Update/Delete Builder 的公共接口）：
- `SqlTemplate` / `Parameters` / `ToSqlString`

### IFastBuilder

快速操作内部构建器（数据库特定实现）：
- `ExecuteBulkCopyAsync` / `UpdateByTempAsync` / `Merge`

### ICustomConditionalFunc

自定义条件扩展点：
- `GetConditionalSql(ConditionalModel, int)` — 条件模型 → SQL

### ISugarDataConverter

自定义数据类型转换器：
- `ParameterConverter<T>` — 写入时参数转换
- `QueryConverter<T>` — 读取时结果转换

## 设计模式总结

| 模式 | 体现 |
|------|------|
| Fluent API | IInsertable/IUpdateable/IDeleteable/IStorageable 链式调用 |
| 多表泛型递进 | ISugarQueryable<T,T2> : ISugarQueryable<T>，最多 12 表 |
| partial interface | ISqlBuilder/ILambdaExpressions 等，跨文件扩展 |
| 同步/异步全覆盖 | 所有终结方法均有 Async 变体 + CancellationToken |
| 工厂 + 命名约定 | InstanceFactory 按 `{DbName}{BaseClass}` 创建数据库特定实现 |
| 仓储标记 | ISugarRepository → ISimpleClient<T>，DI 友好 |

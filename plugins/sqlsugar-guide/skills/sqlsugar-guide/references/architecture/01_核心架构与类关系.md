# SqlSugar 核心架构与类关系

> 基于源码分析，帮助理解内部结构、排查问题、扩展功能。

## 入口类层次

```
SqlSugarScope  ──(每次访问)──>  SqlSugarClient  ──(1:1)──>  SqlSugarProvider
     │                              │                           │
  线程安全，适合 DI              非线程安全                    真正的引擎
  ScopedContext 属性              持有单个 Provider            partial class，实现 ISqlSugarClient
  CallContext 隔离                可切换数据库(ChangeDatabase)  持有 IAdo, AopProvider, QueryFilter 等
```

| 类 | 文件位置 | 职责 |
|---|---|---|
| `SqlSugarScope` | 根目录/SqlSugarScope.cs | DI 友好入口，每次访问创建线程隔离的 Provider |
| `SqlSugarClient` | 根目录/SqlSugarClient.cs | 单连接客户端，1:1 持有 SqlSugarProvider |
| `SqlSugarProvider` | Abstract/SugarProvider/SqlSugarProvider.cs | 真正的引擎，持有所有子系统 |
| `SqlSugarScopeProvider` | Abstract/SugarProvider/SqlSugarScopeProvider.cs | 内部包装，CallContext 线程隔离 |

## 核心子系统一览

```
SqlSugarProvider
  ├─ IAdo (AdoProvider)           — 连接管理、SQL 执行、主从分离
  ├─ Queryable<T>                  — 查询构建器
  ├─ Insertable<T>                 — 插入构建器
  ├─ Updateable<T>                 — 更新构建器
  ├─ Deleteable<T>                 — 删除构建器
  ├─ CodeFirst                     — 代码优先建表/迁移
  ├─ DbFirst                       — 数据库优先生成实体
  ├─ DbMaintenance                 — 数据库维护(表/列/索引)
  ├─ EntityMaintenance             — 实体元数据管理
  ├─ AopProvider                   — AOP 事件系统
  ├─ QueryFilterProvider           — 全局查询过滤器
  ├─ CacheProvider                 — 查询缓存
  ├─ ContextMethods                — 辅助方法(DictionaryToDataTable 等)
  └─ TempItems                     — 请求级临时数据
```

## InstanceFactory — 数据库插件系统

`Infrastructure/InstanceFactory.cs` 是核心工厂，按命名约定创建数据库特定实现：

```
约定: {数据库名}{基类名}

例如 DbType.MySql →
  MySqlQueryable<T>, MySqlQueryBuilder, MySqlBuilder,
  MySqlInsertBuilder, MySqlUpdateBuilder, MySqlDeleteBuilder,
  MySqlDbFirst, MySqlCodeFirst, MySqlDbBind, MySqlDbMaintenance
```

支持自定义数据库插件：
```csharp
InstanceFactory.CustomDllName = "MyDb.SqlSugar.dll";
InstanceFactory.CustomNamespace = "MyDb.SqlSugar";
InstanceFactory.CustomDbName = "MyDb";
// 然后创建 MyDbQueryable<T>, MyDbBuilder 等类
```

## SimpleClient<T> — 内置仓储模式

`SimpleClient.cs` 提供开箱即用的 CRUD：
- `GetById()`, `GetList()`, `GetPageList()`
- `Insert()`, `Update()`, `Delete()`
- `Change<ChangeType>()` — 多态切换实体类型
- 所有方法 virtual，可 override
- 内置 CancellationToken 支持

## Queue 系统 — 批量事务执行

```csharp
db.AddQueue(sql1, params1);
db.AddQueue(sql2, params2);
db.SaveQueues();  // 在一个事务内全部执行
// 或获取查询结果:
db.SaveQueues<T1, T2>();  // 返回 Tuple<List<T1>, List<T2>>
```

## Tracking — 变更追踪

```csharp
db.Tracking(entity);       // 启用追踪
// ... 修改 entity 属性 ...
db.UpdateNav(entity).ExecuteCommand();  // 只更新变更的字段
db.ClearTracking();
```

# SqlSugar 方法调用链

> 理解一次操作从入口到 SQL 执行的完整路径，帮助调试和扩展。

## 查询链路：`db.Queryable<T>().Where(x => x.Id == 1).ToList()`

```
SqlSugarClient.Queryable<T>()
  → SqlSugarProvider.Queryable<T>()
    → InitMappingInfo<T>()              // 从缓存/反射加载 EntityInfo
    → InstanceFactory.GetQueryable<T>() // 创建数据库特定 Queryable
      → new QueryableProvider<T>()      // 设置 Context, SqlBuilder, QueryBuilder
    → UtilMethods.AddDiscriminator()    // 如有鉴别器自动追加 WHERE

.Where(x => x.Id == 1)
  → QueryableProvider<T>.Where(expression)
    → ExpressionContext.Resolve(expression, Where)
      → BaseResolve.Start()             // 表达式访问器分发
        → LambdaExpressionResolve       // 剥离 lambda 体
          → BinaryExpressionResolve     // 处理 == 运算符
            → MemberExpressionResolve   // "x.Id" → "[Id]"
            → ConstantExpressionResolve // "1" → "@Id0"
            → 追加 "[Id] = @Id0" 到 Result
    → QueryBuilder 存储 WHERE 子句

.ToList()
  → QueryableProvider<T>.ToList()
    → SqlBuilder.GetSql()               // 组装完整 SELECT ... FROM ... WHERE ...
    → Ado.GetDataTable(sql, parameters)
      → CheckConnection()               // 按需打开连接
      → ExecuteBefore()                 // 触发 OnLogExecuting AOP
      → ExecuteChangeSql()              // 触发 OnExecutingChangeSql AOP（可改写 SQL）
      → DbCommand.ExecuteReader()       // 实际 ADO.NET 调用
      → GetDataAfter()                  // 触发 OnLogExecuted AOP
    → DbBind.DataReaderToList<T>()      // IDataReader → List<T>
      → IDataReaderEntityBuilder        // Emit 反射加速实体构造
```

## 插入链路：`db.Insertable(data).ExecuteCommand()`

```
SqlSugarClient.Insertable<T>(data)
  → SqlSugarProvider.Insertable<T>(data)
    → InstanceFactory.GetInsertBuilder() // 创建 InsertBuilder
    → new InsertableProvider<T>()
      → 存储 insertObjs, EntityInfo, SqlBuilder, InsertBuilder

.ExecuteCommand()
  → InsertableProvider<T>.ExecuteCommand()
    → _ExecuteCommand()
      → PreToSql()                      // 应用列过滤、IgnoreColumns、InsertColumns
      → InsertBuilder.ToSqlString()     // 生成 "INSERT INTO [table] (...) VALUES (...)"
      → AutoRemoveDataCache()           // 清除相关缓存
      → Ado.ExecuteCommand(sql, params) // 触发 AOP → 执行
      → After()                         // 后置钩子（DiffLog 等）
```

## CodeFirst 链路：`db.CodeFirst.InitTables<MyEntity>()`

```
SqlSugarClient.CodeFirst
  → SqlSugarProvider.CodeFirst (属性)
    → new CodeFirstProvider()

.InitTables<MyEntity>()
  → InitTables(typeof(MyEntity))
    → 检测 SplitTableAttribute → 如有委托给 SplitCodeFirstProvider
    → lock(LockObject)                  // 线程安全
    → Context.GetEntityNoCacheInitMappingInfo(type) // 加载实体元数据
    → DbMaintenance.IsAnySystemTablePermissions()   // 权限检查
    → 无事务时: Ado.UseTran(() => Execute())         // 自动包事务
    → Execute(entityType, entityInfo)
      → 获取现有表结构 (DbMaintenance)
      → 对比实体列与数据库列
      → 生成 ALTER TABLE / CREATE TABLE 语句
      → Ado.ExecuteCommand() 执行 DDL
```

## 表达式解析架构

```
ExpressionContext.Resolve(expression, resolveType)
  │
  ├─ LambdaExpressionResolve       — 剥离 lambda
  ├─ BinaryExpressionResolve       — ==, !=, >, <, &&, ||
  ├─ MethodCallExpressionResolve   — .Contains(), .StartsWith(), SqlFunc.*
  ├─ MemberExpressionResolve       — 属性访问 x.Name
  ├─ ConstantExpressionResolve     — 常量值
  ├─ NewExpressionResolve          — new 构造
  ├─ MemberInitExpressionResolve   — 对象初始化器
  ├─ ConditionalExpressionResolve  — 三元运算符
  ├─ UnaryExpressionResolve        — 一元运算 !, -
  ├─ CoalesceResolveItems          — ?? 运算符
  └─ 导航表达式解析器               — OneToOneNavgateExpression 等
```

每个解析器将表达式片段转换为 SQL 片段和参数，追加到 `ExpressionContext.Result`。
